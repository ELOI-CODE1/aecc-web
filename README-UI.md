# AAEC Ltd UI Change Guide

This document is meant to help you give another AI clear instructions for changing the AAEC Ltd website UI without breaking the existing design direction.

## 1. Project Purpose
The website is a polished, professional, architecture-and-engineering-focused landing experience for AAEC Ltd. It should feel:
- modern
- technical
- trustworthy
- minimal but premium
- suitable for a consulting firm based in Kigali, Rwanda

## 2. What the Current UI Contains
The UI is built as a set of responsive HTML pages using Tailwind CSS and a custom design system.

### Existing pages
- Home
- About
- Services
- Workflow / Project Process
- Contact

### Core sections used across pages
- sticky top navigation
- hero section
- content cards and panels
- blueprint-style background grid
- CTA banners
- structured footer

## 3. Design System to Preserve
When making changes, keep the following visual language consistent.

### Color palette
Use the current brand-inspired palette:
- Primary: dark navy / deep blue
- Secondary: warm amber / orange accent
- Surface: light neutral backgrounds
- Text: dark slate/charcoal tones

### Typography
- Headings: Montserrat
- Body text: Inter
- Keep headings bold, structured, and professional

### Layout style
- Clean spacing
- Strong section separation
- Minimalist but technical feel
- Responsive behavior for mobile and desktop

### Common UI motifs
- blueprint grid background
- subtle technical divider lines
- bordered content panels
- structured lists with dotted leaders
- compact CTA buttons

## 4. Structure Patterns to Reuse
When adding or changing sections, follow these patterns:

### Navigation
- keep a sticky top bar
- keep links simple and minimal
- maintain a clear “Get a Quote” action button

### Hero section
- strong headline
- supporting paragraph
- one or two CTA buttons
- background image or subtle architectural texture

### Content cards
- use bordered panels
- keep consistent padding and spacing
- add hover effects only if subtle

### Footer
- keep company name and short brand description
- include simple policy / location links

## 5. Content Direction for AAEC Ltd
The site should continue to communicate:
- architecture and engineering expertise
- professionalism and trust
- sustainable and modern solutions
- project clarity and workflow reliability
- strong regional presence in Rwanda

## 6. Rules for AI Edits
When asking another AI to change the UI, preserve these principles:
- Do not break the overall architecture of the page
- Keep the same professional tone and premium engineering style
- Preserve the responsive layout
- Maintain the existing color system and typography
- Keep the blueprint/grid motifs unless explicitly removed
- Preserve CTA structure and company branding
- Avoid overly playful or generic web styles

## 7. Safe Change Instructions for Another AI
Use this as a prompt with another AI:

"Update the AAEC Ltd website UI based on the existing design language shown in the provided HTML. Keep the same professional architectural/engineering aesthetic, preserve the current Tailwind-based structure, and maintain the dark navy + warm amber color system, Montserrat/Inter typography, border-based cards, blueprint background motifs, and responsive layout. Update the page for the following request: [insert your change request here]. Keep all existing branding, navigation, and CTA behavior intact unless explicitly requested otherwise."

## 8. Example Change Prompts
### Example 1: Change homepage hero
"Redesign the homepage hero section to feel more premium and modern while keeping the current AAEC brand style. Preserve the existing navbar, button styling, and overall layout, but make the hero more impactful with a stronger headline, better spacing, and a cleaner CTA section."

### Example 2: Add a new services section
"Add a new section to the services page showing additional specializations for environmental consulting, GIS mapping, and quantity surveying. Use the same card-based layout, subtle border styling, and professional tone as the existing sections."

### Example 3: Update the contact page
"Refresh the contact page so it looks more polished and conversion-focused. Keep the same form structure, technical aesthetic, and location/map panel, but improve spacing, hierarchy, and visual clarity."

## 9. What to Avoid
Avoid changing:
- the overall brand identity
- the structured and minimal layout
- the current page purpose
- the professional engineering tone
- the responsive behavior

## 10. Recommended Editing Approach
When changing the UI:
1. identify the target page
2. preserve the existing section hierarchy
3. reuse current Tailwind utility classes where possible
4. keep content concise and professional
5. test the layout on mobile and desktop

This guide is meant to keep future edits aligned with the current AAEC Ltd visual system and content goals.
